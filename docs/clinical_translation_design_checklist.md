# 鍼灸・漢方 × 現代医学：翻訳＝設計チェックリスト（SML/CML版）
> 厳復／西周の「翻訳＝概念設計・媒体設計」の発想を、臨床研究・症例報告・説明文書に落とすための1ページ手順

---

## 0. 使い方（30秒）
- ✅を埋めるだけで「言っていいこと／言えないこと」の境界が固定されます  
- **SML（解釈・記述の整合）**と**CML（価値・責任・秩序）**を分けて書きます  
- 迷ったら「主張レベルの階段（下）」に戻る

---

## 1. 目的関数（CML：何を“良し”とするか）
- [ ] 最優先は **患者安全**（赤旗／紹介／緊急対応）である
- [ ] 主目的を1つに固定：  
  - [ ] 症状軽減（苦痛・頻度・強さ）  
  - [ ] 機能改善（睡眠・活動・仕事・ADL）  
  - [ ] QOL/満足度（PRO）  
  - [ ] 併存症の管理支援（不安、ストレス対処等）
- [ ] 二次目的（あれば）：____________________________________
- [ ] 価値優先順位（例：安全＞生活機能＞症状＞説明の綺麗さ）を明記：  
  - ____________________________________

---

## 2. 対象・読者・媒体（SML/CML：誰に何で伝えるか）
- [ ] 対象（患者像）を特定：年齢／既往／服薬／リスク因子：________________
- [ ] 読者を特定：  
  - [ ] 医師（紹介先） [ ] 患者 [ ] 学会査読者 [ ] 一般向け
- [ ] 媒体を特定（意味が変わるので固定）：  
  - [ ] 症例報告（CARE等） [ ] 研究計画/倫理文書 [ ] 診療録テンプレ  
  - [ ] 患者説明文書 [ ] SNS/ブログ（※主張を弱める）

---

## 3. 用語の「境界設計」（SML：置換禁止・対応は条件付き）
> ここが厳復／西周の核心：単語対応ではなく **“運用定義＋非対応宣言”**

- [ ] 主要用語の運用定義（この文書内だけでもOK）
  - 「証」：____________________________________
  - 「気」：____________________________________
  - 「改善」：____________________________________
  - 「有効」：____________________________________
- [ ] **置換禁止リスト**（例：「肝鬱＝うつ」「気虚＝慢性疲労」を断言しない）
  - [ ] 置換せず併記（例：`肝気鬱結（西洋医学的にはストレス関連症状が示唆される可能性）`）
- [ ] **非対応宣言**（この枠組みで“言えないこと”を先に書く）
  - [ ] 病名の治癒・改善を断言（医師診断＋追跡がない限り）
  - [ ] 機序の証明（症例では仮説に留める）
  - [ ] 因果（本介入が原因だ）を断言

---

## 4. 安全設計（SML→CML：責任境界の明示）
- [ ] 赤旗チェック（該当すれば紹介／受診勧奨を記録）
  - [ ] 神経脱落／麻痺／失禁  
  - [ ] 胸痛・呼吸困難・失神  
  - [ ] 発熱・体重減少・悪性腫瘍疑い  
  - [ ] 妊娠・抗凝固・重篤既往など禁忌/注意  
- [ ] 紹介・連携のルール（誰に、いつ、どう繋ぐか）
  - [ ] 既存主治医：________________
  - [ ] 紹介条件（閾値）：____________________________________
- [ ] **責任境界の宣言**（診断・検査・処方は医師、等）を明記

---

## 5. アウトカム設計（SML：観察可能な形に落とす）
- [ ] 主要アウトカム（1つ）を測れる形で定義：____________________________
- [ ] 測定方法（尺度/頻度/日数/PRO）：__________________________________
- [ ] 観察期間（ベースライン→フォロー）：_______________________________
- [ ] 共介入（薬、運動、生活変化、他治療）を記録：______________________

---

## 6. 介入の記述（SML：再現性ではなく“追跡可能性”）
- [ ] 介入の要点（いつ／何回／どこ／何を狙ったか）を最低限固定
- [ ] 個別調整した部分（なぜ変えたか）を理由つきで残す
- [ ] 施術者の判断の根拠（所見→方針）を短く書く

---

## 7. 主張レベルの階段（SML→CML：飛躍防止）
> 上に行くほど条件が厳しい。症例報告は基本 ①〜②が主戦場。

1. [ ] **観察**：症状/所見/PROが変化した  
2. [ ] **機能**：生活機能が改善した（具体例つき）  
3. [ ] **安全**：赤旗は否定/対応済み、必要時紹介した  
4. [ ] **疾患**：医師診断・検査・追跡により改善が確認された（条件明記）  
5. [ ] **機序**：仮説として提案（“証明”と言わない）

---

## 8. 翻訳の作法（西周型：普及／厳復型：目的偏りを自覚）
- [ ] 読者に合わせて説明層を切る（専門向け／一般向けを混ぜない）
- [ ] 目的偏りの自己申告（例：「症状軽減に焦点、病名改善は扱わない」）
- [ ] 反証可能性を残す（何が起きたら撤回/再評価するか）

---

## 9. 透明性・監査可能性（EVA的：Accountability）
- [ ] 同意（症例報告／情報公開／AI使用）を明記
- [ ] 利益相反・限界・代替説明（自然経過、回帰、プラセボ等）を記載
- [ ] 「何を保存したか」（記録・尺度・紹介記録）を列挙

---

## 10. 最終チェック（1行で通るか）
- [ ] この文章は「医師に読まれても責任境界が明確」  
- [ ] この文章は「患者に読まれても誤解（病名治癒の断言）が起きにくい」  
- [ ] この文章は「SML（記述）とCML（価値・責任）が混線していない」

---

### 付記：1行テンプレ（先頭に置くと事故が減る）
> 本報告は、（主目的）________ を（評価指標）________ で追跡し、赤旗対応と紹介基準を明示したうえで、疾患治癒や機序の断言は行わない。
